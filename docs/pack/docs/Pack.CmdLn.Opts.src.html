128 | ||| Names of all command line options (prefixed with "-" in case of
129 | ||| single-character option names and with "--" in case of multi-character

138 | ||| Given the current directory (from which pack was invoked)
139 | ||| and an initial config assembled from the `pack.toml` files
140 | ||| in scope, generates the application
141 | ||| config and command to run from a list of command
142 | ||| line arguments.
143 | |||
144 | ||| @ c      : Type representing the command to run
145 | |||            We abstract over this type, because pack and
146 | |||            pack-admin accept different kinds of commands,
147 | |||            and both applications use this function to parse
148 | |||            the command line args
149 | |||
150 | ||| @ curDir : Current working directory
151 | ||| @ init   : Initial config (possibly assebled from `pack.toml` files)
152 | ||| @ args   : List of command line arguments
153 | export
154 | applyArgs :  (0 c : Type)
155 |           -> Command c
156 |           => (curDir     : CurDir)
157 |           -> (init       : MetaConfig)
158 |           -> (args       : List String)
159 |           -> Either PackErr (MetaConfig, c)
160 | applyArgs c dir init args =
161 |   case getOpt RequireOrder descs args of
162 |        MkResult opts n  []      []       => do
163 |          cmd  <- readCommand c dir n
164 |          let init' = {logLevel := defaultLevel cmd} init
165 |          conf <- foldlM (\c,f => f dir c) init' opts
166 |          Right (conf, cmd)
167 |
168 |        MkResult _    _ (u :: _) _        => Left (UnknownArg u)
